System Overview

The Inventory Management System (IMS) is designed to streamline the procurement process through structured workflows for tender management, vendor selection, and inventory tracking.

Core Purpose: Automate and track the complete lifecycle of procurement tenders, from creation through vendor selection, approval, and finalization into inventory.

User Roles & Permissions

1. IMS Super Admin

Access Level: Full system access across all offices and wings

Permissions:

  • Create, edit, delete tenders
  • View all tenders from all wings
  • Manage vendors (add, edit, view)
  • Manage users and permissions
  • Access reports and analytics
  • Finalize tenders to stock acquisition system

2. IMS Admin

Access Level: Office-scoped access

Permissions:

  • Create and manage tenders for assigned office
  • View vendors and manage bidders
  • Edit tender status and details
  • Finalize tenders
  • View office-specific reports

3. Wing Supervisor

Access Level: Wing-scoped access

Permissions:

  • View tenders for their wing
  • Manage inventory within their wing
  • Track tender status
  • Access wing-specific reports

4. General User

Access Level: Read-only or limited access

Permissions:

  • View assigned tenders
  • Track inventory items
  • Generate reports (read-only)

ğŸ”’ Finalized Tenders

Once a tender is finalized, it becomes read-only. Users can only view, print, and export the data. No edits or deletions are allowed. This ensures audit trail and data integrity.
