Branch Request to Tender and Final Allotment Workflow

Simple diagram of the branch request flow from employee request to storekeeper check, supervisor approval, tender demand, procurement, and final allotment.

flowchart TD
    A[Branch Employee creates request] --> B[Branch Storekeeper receives request]

    B --> C[Storekeeper checks branch stock]
    C --> D{Item available?}

    D -->|Yes / Full or partial available| E[Storekeeper marks available quantity]
    E --> F[Forward to Branch Supervisor]
    F --> G{Branch Supervisor decision}

    G -->|Approved| H[Branch Storekeeper allots available item to employee]
    G -->|Rejected / Returned| Z[Request rejected or returned with comments]

    H --> I[Stock deducted from branch/store inventory]
    I --> J[Item added to employee inventory]
    J --> K[Request completed]

    D -->|No / Short quantity| L[Storekeeper marks shortage quantity]
    L --> F

    G -->|Approved but item not available| M[Shortage becomes Required Item / Demand]
    M --> N[Admin workflow starts by item group]

    N --> O[DD Admin review]
    O --> P[AD Admin review]
    P --> Q{Admin approved?}

    Q -->|No| Z
    Q -->|Yes| R[Demand goes to Tender Module]

    R --> S[Required Items Queue]
    S --> T[Procurement groups similar demands]
    T --> U[Assign demand to new or existing tender]
    U --> V[Tender items prepared]
    V --> W[Vendor / PO / Delivery / Stock Acquisition]

    W --> X[Stock received]
    X --> Y[Storekeeper allots procured item to employee]
    Y --> I

Simple Flow

Employee Request
   ↓
Branch Storekeeper checks stock
   ↓
Branch Supervisor approves / rejects
   ↓
If available:
   Branch Storekeeper allots item to employee
   ↓
   Inventory updated

If not available:
   Required Item / Demand created
   ↓
   Admin workflow by item group
   ↓
   Tender module receives demand
   ↓
   Tender / PO / Delivery / Stock Acquisition
   ↓
   Storekeeper allots item to employee
   ↓
   Inventory updated
Main idea: Branch Storekeeper checks availability first. Branch Supervisor approves the need. Available items are allotted directly. Unavailable items become demand for tender. After procurement, the item is allotted to the employee.
