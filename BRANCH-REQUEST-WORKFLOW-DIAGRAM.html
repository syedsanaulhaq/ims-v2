Branch Request to Tender and Final Allotment Workflow

Simple diagram showing that procured/admin stock is allotted to the Branch Supervisor first, handed to Branch Storekeeper, entered into branch store, and then issued to the requested employee.

flowchart LR
    subgraph S1[Branch Request Starts]
      A[Branch Employee creates request]
      B[Branch Storekeeper receives request]
      C[Storekeeper checks branch stock]
      D{Item available?}
      A --> B --> C --> D
    end

    subgraph S2[Branch Approval]
      E[Storekeeper marks available quantity]
      L[Storekeeper marks shortage quantity]
      F[Forward to Branch Supervisor]
      G{Branch Supervisor decision}
      Z[Request rejected or returned with comments]
      E --> F
      L --> F
      F --> G
      G -->|Rejected / Returned| Z
    end

    subgraph S3[Available in Branch Store]
      H[Branch Storekeeper issues available item to requested employee]
      I[Stock deducted from branch store]
      J[Item added to employee inventory]
      K[Request completed]
      H --> I --> J --> K
    end

    subgraph S4[Not Available: Admin and Tender Demand]
      M[Shortage becomes Required Item / Demand]
      N[Admin workflow starts by item group]
      O[DD Admin review]
      P[AD Admin review]
      Q{Admin approved?}
      R[Demand goes to Tender Module]
      S[Required Items Queue]
      T[Procurement groups similar demands]
      U[Assign demand to new or existing tender]
      V[Tender items prepared]
      W[Vendor / PO / Delivery / Stock Acquisition]
      M --> N --> O --> P --> Q
      Q -->|Yes| R --> S --> T --> U --> V --> W
      Q -->|No| Z
    end

    subgraph S5[Procured Stock Handoff to Branch]
      X[Stock received by Admin / Central Store]
      Y[Admin allots approved stock to Branch Supervisor]
      AA[Branch Supervisor gives stock to Branch Storekeeper]
      AB[Branch Storekeeper enters item into branch store]
      AC[Branch Storekeeper issues item to requested employee]
      X --> Y --> AA --> AB --> AC
    end

    D -->|Yes / full or partial available| E
    D -->|No / short quantity| L
    G -->|Approved and available| H
    G -->|Approved but not available| M
    W --> X
    AC --> I

Simple Flow

Employee Request
   ↓
Branch Storekeeper checks stock
   ↓
Branch Supervisor approves / rejects
   ↓
If available:
   Branch Storekeeper allots item to employee
   ↓
   Inventory updated

If not available:
   Required Item / Demand created
   ↓
   Admin workflow by item group
   ↓
   Tender module receives demand
   ↓
   Tender / PO / Delivery / Stock Acquisition
   ↓
  Admin / Central Store receives stock
  ↓
  Admin allots stock to Branch Supervisor
  ↓
  Branch Supervisor gives stock to Branch Storekeeper
  ↓
  Branch Storekeeper enters item into branch store
  ↓
  Branch Storekeeper issues item to requested employee
   ↓
   Inventory updated
Main idea: Branch Storekeeper checks availability first. Branch Supervisor approves the need. Available items are issued directly from branch store. Unavailable items become tender demand; after procurement, Admin allots stock to the Branch Supervisor, the Branch Supervisor gives it to the Branch Storekeeper, and the Branch Storekeeper issues it to the requested employee.
