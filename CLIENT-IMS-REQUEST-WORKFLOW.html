Inventory Request to Final Issuance Workflow

2. Supervisor Decision Point

Path A: Item Available in Branch/Wing Stock

The supervisor does not need admin procurement/workflow approval. The request goes directly to the local storekeeper for issuance.

Employee Request→ Supervisor Review→ Qty Adjusted + Comments→ Storekeeper Issues→ Requester Inventory Updated

Path B: Item Not Available in Branch/Wing Stock

The supervisor forwards the request to admin workflow. The workflow is based on the group of the requested item.

Employee Request→ Supervisor Review→ Forward to Admin→ DD Admin→ AD Admin→ Storekeeper
Important: At every approval stage, the approver should be able to change the approved quantity and add comments. The system should preserve the requested quantity, approved quantity, approver name, date/time, and comments for audit/history.

3. Group-Wise Admin Workflow

When stock is not available locally, the request is routed according to the group of the requested item. Each group can have its own approval path. A common client-friendly example is:

Supervisor Forward→ DD Admin→ AD Admin-I / AD Admin-II→ Storekeeper→ Issue Item
Stage Who Acts What They Can Do System Output
Request Submission Employee Select item, enter quantity, write purpose/justification. Request number generated and request status becomes Submitted/Pending.
First Review Supervisor Check stock, approve/reject/forward, change quantity, write comments. Request either goes to storekeeper or admin workflow.
Admin Group Workflow DD Admin / AD Admin Review item group, change approved quantity, approve/reject/forward, write comments. Approval history is updated and request moves to next admin step.
Issuance Storekeeper Verify final approved item and quantity, issue stock to requester. Stock is deducted from store inventory and added to requester inventory.
Completion System Record final issued quantity, issuer, issue date, and inventory ledger movement. Request status becomes Issued/Completed and appears in history/reports.

4. Final Inventory Movement

Store Inventory

Deduct: Final issued quantity is subtracted from the issuing storekeeper's stock.

Example: 20 available - 3 issued = 17 remaining.

→

Stock Ledger

The system records a transaction with request number, item, quantity, issuer, receiver, date, and remarks.

→

Requester Inventory

Add: The issued item is added to the employee's personal inventory or branch/wing possession list.

The requester can later see it in My Inventory / Issued Items.

5. End-to-End Example

  1. Employee of a branch requests 5 laptops.
  2. Supervisor receives the request and checks local branch stock.
  3. If 3 laptops are available, supervisor may change approved quantity from 5 to 3 and add comments.
  4. If local stock is enough, supervisor sends the request to the branch/wing storekeeper for issuance.
  5. If local stock is not enough, supervisor forwards it to admin workflow based on the laptop item group.
  6. DD Admin and AD Admin review the request, can adjust quantity, and record comments.
  7. After final approval, storekeeper issues the approved quantity.
  8. IMS deducts the issued quantity from store inventory, writes stock transaction history, and adds the item to the requester inventory.
